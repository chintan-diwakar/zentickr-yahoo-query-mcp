# Zentickr v0.1.0 Release â€” Design

**Date:** 2026-07-17
**Goal:** Turn zentickr-yahoo-query-mcp into a production-quality open-source MCP
server and ship v0.1.0 as a GitHub Release (no PyPI), with contribution
infrastructure so outside developers can build on it and raise PRs.

## Decisions

- **Release target:** GitHub Release only (tag `v0.1.0` + release notes). PyPI later if wanted.
- **Naming:** consolidate on `zentickr` everywhere â€” package name, module dir
  (`src/zentickr/`), CLI command. The old `stock_mcp` / `stock-mcp-server` /
  leftover "alex-mcp"/"OpenAlex" strings go away.
- **Scope:** lean production release. Fix every audit finding, add offline tests +
  CI, rewrite docs, add contributor kit. No speculative features (no live-API
  test suite, Docker, retries, or release automation in v0.1.0).

## Current state (audited highlights)

- FastMCP server in `src/stock_mcp/server.py` (~430 lines) exposing ~22
  yahooquery-backed tools. MIT license, public repo.
- **Fatal:** metadata file is `project.toml`, not `pyproject.toml` â€” the package
  was never installable and the declared console script never existed.
- **Broken entry point:** `main()` wraps synchronous `mcp.run()` in
  `asyncio.run()`; duplicate `if __name__ == "__main__"` blocks; template

- **Dependency drift:** `requirements.txt` vs `project.toml` disagree; toml
  missing `mcp`, lists likely-unused `fastapi`/`uvicorn`/`rich`/`httpx`.
- **Serialization bug:** `pd.isna(data)` raises `ValueError` for array-likes in
  `convert_to_json_serializable`.
- **No error handling** in tools; identical 3-line ticker-parsing block repeated ~20Ă—.
- **No tests, no CI, no contributor docs.** README contains invented sample
  output and placeholder URLs.
- Unreviewed helper scripts: `deploy.py`, `validate.py`, `setup_dev.py` â€”
  default disposition is delete (YAGNI; git remembers).

## Phase 1 â€” Audit (Fable)

Full code audit of the repo (server correctness, packaging, scripts, docs
consistency). Output: a findings list with severity, each mapped to a fix in
Phase 2 or explicitly waived. The known findings above are the seed; the audit
confirms them and hunts for anything missed (e.g., blanket `except`, index
handling in `history()` output, `.gitignore` gaps).

## Phase 6 â€” Release

- Version `0.1.0` in `pyproject.toml` and `__init__.py`, commit, push, tag

## Error handling policy

Tools never leak tracebacks to MCP clients: yahooquery/network failures and bad
symbols produce a short, human-readable error string identifying the tool and
symbol(s). The serialization helper handles pandas objects, NaN, and array-likes
without raising.

## Testing policy

Unit tests are offline and deterministic (mock yahooquery). Live behavior is
verified manually before tagging the release (Phase 6). No live-network tests in
CI for v0.1.0.

## Non-goals (v0.1.0)

PyPI publishing, Docker, streaming/real-time data, retries/rate-limit handling,
options/crypto/news tools, release automation, docs site.
